# Authentication & HTTP Context

When working on any source file in this project, be aware of the authentication system:

## Key Files

- **Token storage**: `src/utils/TokenManager.js` — singleton, stores `{ authenticated, timestamp }` in `sessionStorage` under key `esp32_auth_meta`
- **HTTP client**: `src/utils/HttpClient.js` — singleton, cookie-based auth (`credentials: 'same-origin'`), dispatches `auth-error` events on 401/403
- **HTTP agent**: `src/utils/HttpRequestAgent.js` — singleton, higher-level wrapper with `login()`, `logout()`, `validateToken()`, `get()`, `post()` methods
- **Auth state**: `src/features/Auth/AuthSlice.js` — Redux slice with `loginUser`, `logoutUser`, `initializeAuth` thunks; stores `user.role` (`"service"` or `"client"`)
- **Route guard**: `src/features/Auth/components/AuthGuard.jsx` — wraps entire app, validates token on startup
- **Middleware**: `src/middleware/authMiddleware.js` — handles `auth-error` events and session expiration checks
- **API config**: `src/config/apiConfig.js` — all endpoints (including `SYSTEM_CREDENTIALS`), public endpoint list, error messages
- **Types**: `src/types/auth.js` — JSDoc types: `AuthState`, `UserInfo`, `LoginCredentials`, `TokenData`
- **Credential service**: `src/services/CredentialService.js` — `getCredentials()`, `saveCredentials()`, `deleteCredentials()` for client account management (service-role only)

## Patterns to Follow

- Use `HttpRequestAgent` for API calls (not `HttpClient` directly) in services and thunks
- Use `skipAuth: true` option for public endpoints (login, health, status, wifi)
- Always handle `AbortError` separately when using abort signals
- Clear tokens on any auth failure to maintain clean state
- Use `ErrorHandlingService` for user-facing error messages
- Redux thunks should use `rejectWithValue()` for error propagation
- Check `user.role === 'service'` before rendering credential management UI
- Use `CredentialService` (not `HttpRequestAgent`) for credential CRUD — it uses `HttpClient.delete()` directly
